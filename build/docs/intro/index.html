个人技术文档库

欢迎来到我的个人技术知识库！这里主要用于记录在开发、系统设计、工具探索过程中的技术沉淀与日常心得。

:::tip 沉淀与分享 “好记性不如烂笔头。” 记录每一次技术攻关和踩坑经历，不仅能帮助自己快速回顾，还能为未来的项目提供宝贵的参考。 :::

📂 知识库架构​

当前知识库主要包含以下几个核心板块，你可以通过左侧侧边栏进行探索：

1. ⚙️ 核心技术与开发 (/docs/dev)​

记录核心开发技术、IDE 工具链使用指南、项目脚手架与架构设计。

  • 🎯 IDE 工具：如 Antigravity-IDE 的高效使用与 AI 编程实践。
  • 🏗️ 架构实践：模块化设计、系统高可用与微服务演进。

2. 🌐 前端生态与工具​

关注现代 Web 开发技术，探索前端的最佳实践。

  • ⚡ Docusaurus：静态站点生成、MDX 动态组件、内容国际化 (i18n)。
  • 🎨 UI/UX 动效：Sleek Gradients, HSL 调色系统与微动效交互设计。

3. 🛡️ 运维与 DevOps​

记录自动化部署、容器化技术以及云原生基础设施。

  • 📦 Docker & K8s：容器化打包、本地集群调试与容器网络优化。
  • 🔄 CI/CD Pipeline：基于 GitHub Actions 或 Gitlab 的自动化构建流。

✍️ 如何新增与管理文档​

本知识库基于 Docusaurus 与 MDX 驱动。你可以非常轻松地添加新内容：

新建文档步骤​

  1. 创建 Markdown/MDX 文件： 在 docs/ 目录下（或其子目录下）新建一个 .md 或 .mdx 文件。例如：

    touch docs/dev/my-new-tech.md

  2. 添加 Front Matter（页头元数据）： 在文件最顶部添加 YAML 格式的配置项，用于控制标题、排序等：

    ---
    id: my-new-tech
    title: 我的新技术记录
    sidebar_position: 2
    ---

  3. 编写内容： 你可以使用标准 Markdown，也可以使用 MDX 特有的 React 组件或高级样式。

MDX 进阶特性示例​

1. 交互式提示框 (Admonitions)​

可以使用 ::: 来包裹提示框，Docusaurus 会自动渲染为精美的样式：

:::info 提示
这是一个普通提示框，用于补充背景说明。
:::

:::danger 警告
这是一个高风险操作提示，例如：请勿在生产环境直接运行危险命令。
:::

2. 代码高亮与交互​

支持主流编程语言的语法高亮，并带有便捷的“复制”按钮：

// 示例：快速启动 Express 服务的代码片段
const express = require('express');
const app = express();
const PORT = 3000;

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});
