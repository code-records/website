OverlayWindow 如何「传到」TestGame.exe（架构说明）

本文说明 OverlayWindow 并不会作为 Qt 对象进入 TestGame；跨进程传递的是 像素、输入消息，以及（可选）语义通道上的控件树 JSON。阅读时建议先看 结论，再按需跳章节。

OverlayDll（DXGI Present 钩子、每帧 RenderOverlay、共享内存读入与 D3D 合成）的逐项说明见 OVERLAY_DLL.md。

结论（先讲清楚）

  • OverlayWindow（QMainWindow）只存在于 OverlayUI.exe 进程，不会序列化、也不会被 TestGame 加载。
  • TestGame 里看到的「Qt 界面」是：同一套布局在 OverlayUI 里 grab() 成 QImage → 写入共享内存 → OverlayDll 读出来当 D3D 纹理画到游戏画面上。
  • 反向：你在游戏窗口上点的鼠标，由 TestGame 的 WndProc 换算成面板坐标，经 命名管道 发回 OverlayUI，OverlayWindow 里 sendEvent 注入 QMouseEvent，按钮才能响应。
  • 语义（控件树等）：若需要结构化 UI 信息，demo 里另有 命名管道 GameOverlaySemantics（OverlaySemanticsWireHeader + JSON），与位图通道分离；详见 common/overlay_ipc.h 与 TestGame/semantics_reader.cpp。

文字版（与上图等价）

  1. OverlayUI：定时器触发 → 对 panel_ 做 grab() → 得到 QImage。
  2. OverlayUI：把图像写入 共享内存（头 + RGBA 像素，互斥量保护）。
  3. TestGame：LoadLibrary(OverlayDll.dll) → 钩住 Present。
  4. OverlayDll：每帧 Present 前从共享内存 读像素 → 上传 D3D 纹理 → 在交换链上画 居中矩形（与面板同逻辑尺寸）。
  5. 鼠标：TestGame 窗口收到鼠标消息 → 换算成面板局部坐标 → 管道发给 OverlayUI → sendEvent 交给具体子控件。

1. 像素路径（UI「长什么样」进游戏）

步骤 位置 作用
定时抓帧 OverlayUI/overlay_window.cpp onTimer() panel_->grab().toImage()，得到与 panel_ 同尺寸的位图（含 QSS 样式）
写入共享内存 同上 writeFrame() + ensureMapping() CreateFileMappingA 名 GameOverlaySharedMem，头部 SharedOverlayHeader，紧接 RGBA 像素；用 GameOverlaySharedMutex 保护
读出并绘制 OverlayDll/overlay_renderer.cpp RenderOverlay() → OverlayDll/shared_surface.cpp LockForRead OpenFileMapping / MapViewOfFile(...,0,0,0) 映射整块；拷到动态纹理；用 居中像素矩形 + 点采样画到当前 swap chain 的后备缓冲（在 Present 之前）

约定尺寸与布局来自 common/overlay_ipc.h 的 kOverlayPanelWidth / kOverlayPanelHeight 与 kOverlaySharedMappingBytes，OverlayUI 与 DLL 必须一致。

2. 输入路径（游戏里点按钮）

步骤 位置 作用
命中检测与坐标换算 TestGame/main.cpp WM_*MOUSE* 用 MapGameClientToOverlayPanel / ...Clamped（与 DLL 居中矩形同一套数学）得到 面板局部 (x,y)
发到管道 同上 SendOverlayInputToQt() WriteFile 到 \\.\pipe\GameOverlayPipe，载荷 IpcInputEventMsg（move / left down / left up 等）
接收并注入 Qt OverlayUI/overlay_window.cpp onPipeReadyRead → deliverPanelInput panel_->childAt 找到子控件，QCoreApplication::sendEvent 发 合成鼠标事件

因此：不是 OverlayWindow 被传过去，而是 「游戏里的屏幕坐标 → 管道 → Qt 里模拟鼠标」。

3. 进程与加载关系

  • TestGame LoadLibrary OverlayDll.dll（与 exe 同目录），Overlay_Start 安装 DXGI Present / Present1 钩子，在每次 Present 前调用 RenderOverlay。细节见 OVERLAY_DLL.md 第 9 节。
  • OverlayUI.exe 可由 TestGame CreateProcess 拉起（同目录 OverlayUI.exe），也可手动开；只要 共享内存 + 管道 能连上，逻辑就成立。
组件 谁启动 说明
OverlayDll.dll TestGame LoadLibrary（与 exe 同目录） Overlay_Start → Present 钩子里调用 RenderOverlay。
OverlayUI.exe TestGame 可 CreateProcess 拉起，也可手动开 生产像素与输入服务；语义管道由 OverlayUI 侧 QLocalServer 提供。

4. 常见误解对照

误解 实际情况
把 OverlayWindow 或 QWidget 指针交给 TestGame 跨进程只有 字节流（像素 + 小消息）；TestGame 侧只有 D3D 和 Win32，没有 Qt 对象指针。
游戏窗口里嵌了真 Qt 子窗口 是 全屏游戏 HWND + 每帧贴一张与 Qt 同步的纹理，看起来像叠层。

5. 扩展与其它 IPC

若不仅要位图，还要 控件树、布局元数据、可访问性信息 等，一般需要 单独协议（结构化序列化 + 与帧同步的序号）或改为 进程内嵌 Qt——那是另一条架构路线。

本仓库 demo 已包含 独立语义管道 GameOverlaySemantics：OverlayUI 在帧更新后推送 OverlaySemanticsWireHeader + JSON 节点列表；TestGame 侧可用 semantics_reader 线程消费（调试输出）。OverlayDll 不解析该 JSON，仅负责位图合成。协议字段见 common/overlay_ipc.h。
